Exploratory Data Analysis (EDA)

Ecommerce Customer Behavior Analysis

1. Project Introduction

In todays data-driven business environment, understanding customer behavior is essential for making informed decisions related to marketing, inventory management, pricing, and customer retention. This project focuses on performing Exploratory Data Analysis (EDA) on a customer purchase behavior dataset to uncover meaningful patterns, trends, and insights.

The dataset contains information related to customer demographics, product details, purchase history, payment preferences, shipping choices, promotional usage, and customer feedback. However, like most real-world datasets, the raw data contained missing values, inconsistent formats, invalid entries, and noise that could negatively impact analysis.

To address these issues, a structured data cleaning and preparation process was implemented before conducting EDA. Once the data was cleaned, various statistical summaries and visualizations were used to analyze customer behavior and purchasing trends.

2. Project Objectives

The main objectives of this project are:

  • To clean and preprocess raw customer purchase data

  • To handle missing, inconsistent, and invalid values effectively

  • To understand customer demographics and buying behavior

  • To analyze product performance and seasonal trends

  • To study the impact of discounts, promo codes, and subscriptions

  • To identify customer loyalty patterns

  • To visualize insights using Matplotlib and Seaborn

3. Dataset Source

The dataset used in this project was obtained from an external online source for educational and analytical purposes. It contains information related to customer demographics, product details, purchase history, payment methods, shipping preferences, promotional usage, and customer reviews.

4. Scope of the Analysis

This analysis is limited to exploratory insights and visualization-based observations. No predictive modeling is applied in this project. The results of this EDA can be further used as a foundation for advanced analytics or machine learning tasks such as customer segmentation, churn prediction, or sales forecasting.

In[663]:
categorical_cols = [
    'item_purchased', 'size', 'color', 'season',
    'subscription_status', 'shipping_type',
    'discount_applied', 'promo_code_used',
    'frequency_of_purchases', 'gender',
    'category', 'location', 'payment_method'
]

for col in categorical_cols:
    data[col] = data[col].astype(str).str.strip().str.title()
    data[col].replace(['Nan', 'Na', 'None', '', 'N/A'], np.nan, inplace=True)
    data[col].fillna('Unknown', inplace=True)

  • Categorical column selection: Groups all nominal and ordinal features that need consistent formatting and missing-value handling.

  • Convert to string: Ensures uniform processing even if columns contain mixed data types.

  • str.strip(): Removes leading and trailing spaces to avoid duplicate categories (e.g., " Male" vs "Male").

  • str.title(): Standardizes text casing so categories look consistent (e.g., male Male, credit card Credit Card).

  • Replace invalid placeholders with NaN: Treats values like 'Nan', 'Na', 'None', empty strings, and 'N/A' as missing data.

  • Fill missing values with 'Unknown': Preserves rows while clearly labeling missing categorical information.

  • Result: All categorical columns are clean, standardized, free from noisy placeholders, and ready for grouping, visualization, and modeling.

  • Convert to uppercase: str.upper() standardizes size values (s, m, xl S, M, XL) to avoid duplicate categories.

  • Define valid sizes: Creates a clear rule set for acceptable size values (S, M, L, XL).

  • Filter invalid entries: Keeps only rows with valid sizes and removes inconsistent or noisy data (e.g., XS, XXL, typos).

  • Result: The size column becomes clean, standardized, and limited to meaningful categories, improving data quality for analysis and visualization.

Project Conclusion

This project focused on performing a comprehensive Exploratory Data Analysis (EDA) on a customer purchase behavior dataset, with the objective of transforming raw, unstructured data into meaningful and actionable insights.

The first phase of the project involved data cleaning and preprocessing, where issues such as missing values, inconsistent data formats, invalid entries, and noisy categorical values were systematically addressed. Numerical columns were converted to appropriate data types, missing values were handled using statistical methods such as mean imputation, and invalid ranges were removed. Categorical variables were standardized by fixing case inconsistencies, trimming whitespace, and removing unknown or incorrect values. This step significantly improved the overall data quality and reliability of the dataset.

After cleaning, exploratory data analysis was conducted to understand customer demographics, purchase behavior, product preferences, and engagement patterns. The analysis revealed clear trends in customer age distribution, gender-based purchasing behavior, product popularity, size demand, seasonal variations, and category performance. Additionally, customer loyalty indicators such as previous purchases, subscription status, and frequency of purchases provided valuable insights into repeat buying behavior.

Data visualization using Matplotlib and Seaborn played a crucial role in uncovering these patterns. Visual tools such as histograms, bar charts, scatter plots, and box plots made it easier to identify distributions, relationships, and outliers. Visual analysis also helped in understanding the impact of discounts, promo codes, payment methods, and shipping preferences on customer purchasing decisions. Review rating analysis further highlighted overall customer satisfaction and its relationship with spending behavior.

Overall, this project demonstrates the importance of a structured data cleaning process followed by effective EDA and visualization techniques. The cleaned and analyzed dataset provides a strong foundation for data-driven decision-making and can be further extended for advanced analytics or machine learning applications such as customer segmentation, churn prediction, or sales forecasting.
